# Document creation and task management guidelines
_Exported on 2025. 12. 2. at 20시 3분 45초 GMT+9 from Cursor (2.1.46)_

---

**User**

@docs/req.md 에 나타난 사용자 시나리오, 개발 단계, 기능 정의에 따라

user story 명세문서(template:@docs/USER_STORY_TEMPLATE.md )와  기능 명세문서(template:@docs/FEATURE_SPEC_TEMPLATE.md )를 작성하려고해.

@docs/UI_DESIGN_GUIDE.md 과 @docs/DEVELOPMENT_GUIDE.md 에 맞추어 작성하면 돼. 단계별 개발 진행을 위해, 각 문서는 개별 문서로 작성되어야 하며, 해당 명세에 대한 개발을 진행하기 위한 추가 문서/자료가 링크되어 있어야해. 최소 개발 단위는 "기능명세" 단위야

많은 작업이 될 수 있어서, 작성해야하는 문서의 항목들과 작성하기 위한 지시사항들을 정의하고, 문서단위로 작업 상태를 관리하며 작업 상태를 저장 했다가, 새로운 세션에서 저장된 상태로부터 완전히 이어서 할 수 있도록 cursor command 를 먼저 작성해줘

작업상태등의 관리를 위한 저장 경로 : .snapshot/{command_name}

cursor command 저장 경로 : .cursor/commands/{command_name}.md

**Cursor**


요구사항과 가이드 문서를 확인 중입니다. 이후 Cursor command를 작성하겠습니다.



요구사항과 가이드 문서를 확인했습니다. Cursor command를 작성해 작업 상태를 관리하고 세션 간 이어서 진행할 수 있도록 구성 중입니다.



요구사항을 분석해 문서 목록을 정리하고, Cursor command와 작업 상태 관리 시스템을 구성 중입니다.


```markdown
# Generate Specification Documents

## 개요
`docs/req.md`에 정의된 사용자 시나리오, 개발 단계, 기능 정의에 따라 User Story 명세문서와 기능 명세문서를 작성하는 명령어입니다.

## 사용법
```
@generate-spec-documents [옵션]
```

### 옵션
- `--status`: 현재 작업 상태 확인
- `--continue`: 다음 문서 작성 계속 진행
- `--reset`: 작업 상태 초기화 (주의: 모든 진행 상황이 삭제됩니다)
- `--list`: 작성할 문서 목록 확인

## 작업 흐름

1. **초기화**: 작업 상태 파일 확인 및 생성
2. **문서 목록 생성**: `docs/req.md` 분석하여 작성할 문서 목록 생성
3. **문서 작성**: 템플릿에 따라 각 문서 작성
4. **상태 저장**: 작업 진행 상황 저장

## 작업 상태 관리

작업 상태는 `.snapshot/generate-spec-documents/` 디렉토리에 저장됩니다:
- `status.json`: 전체 작업 상태
- `documents.json`: 문서 목록 및 상태
- `current.json`: 현재 작업 중인 문서 정보

## 문서 구조

### User Story 문서
- 경로: `docs/user_stories/US-X.X-[feature-name].md`
- 템플릿: `docs/USER_STORY_TEMPLATE.md`

### Feature Spec 문서
- 경로: `docs/feature_specs/FS-X.X-[feature-name].md`
- 템플릿: `docs/FEATURE_SPEC_TEMPLATE.md`

## 문서 작성 우선순위

Phase별 우선순위에 따라 작성:
1. Phase 1 (P0): 필수 기능
2. Phase 2 (P0): 사용자 설정 기능
3. Phase 3: 기능 개선
4. Phase 4: 고급 기능 (선택사항)

```

```markdown
# 문서 작성 지시사항

## 개요
이 문서는 `docs/req.md`에 정의된 요구사항에 따라 User Story와 Feature Spec 문서를 작성하기 위한 지시사항을 포함합니다.

## 문서 작성 순서

### Phase 1: 기본 기능 구현 (MVP) - 우선순위: P0
1. **US-1.1 / FS-1.1**: 현재 위치 기반 날씨 정보 조회
2. **US-1.2 / FS-1.2**: 날씨 기반 장소 추천 (기본)
3. **US-1.3 / FS-1.3**: 기본 UI 구현

### Phase 2: 사용자 설정 기능 - 우선순위: P0
4. **US-2.1 / FS-2.1**: 지역 선택 기능
5. **US-2.2 / FS-2.2**: 추천 개수 설정
6. **US-2.3 / FS-2.3**: 거리/지역 필터 설정

### Phase 3: 기능 개선 - 우선순위: P1
7. **US-3.1 / FS-3.1**: 장소 상세 정보 표시
8. **US-3.2 / FS-3.2**: 장소 사진 표시
9. **US-3.3 / FS-3.3**: UI/UX 개선

### Phase 4: 고급 기능 (선택사항) - 우선순위: P2
10. **US-4.1 / FS-4.1**: 지도 표시
11. **US-4.2 / FS-4.2**: 날씨 예보 기반 추천
12. **US-4.3 / FS-4.3**: 사용자 선호도 설정

## 문서 작성 규칙

### User Story 작성 시
1. `docs/USER_STORY_TEMPLATE.md` 템플릿 사용
2. `docs/req.md`의 해당 기능 요구사항(FR-X.X) 참조
3. 사용자 시나리오 섹션에 `docs/req.md`의 시나리오 반영
4. 관련 Feature Spec 문서 링크 포함
5. 관련 API 문서 링크 포함 (해당하는 경우)

### Feature Spec 작성 시
1. `docs/FEATURE_SPEC_TEMPLATE.md` 템플릿 사용
2. `docs/req.md`의 해당 기능 요구사항(FR-X.X) 참조
3. `docs/DEVELOPMENT_GUIDE.md`의 개발 가이드 준수
4. `docs/UI_DESIGN_GUIDE.md`의 UI 디자인 가이드 준수
5. 관련 User Story 문서 링크 포함
6. 관련 API 문서 링크 포함
7. 구현 상세에 TypeScript 타입 정의 포함
8. UI/UX 설계에 와이어프레임 및 사용자 플로우 포함

## 참고 문서

## 개요
`docs/req.md`에 정의된 사용자 시나리오, 개발 단계, 기능 정의에 따라 User Story 명세문서와 기능 명세문서를 작성하는 명령어입니다.

## 사용법
```
@generate-spec-documents [옵션]
```

### 옵션
- `--status`: 현재 작업 상태 확인
- `--continue`: 다음 문서 작성 계속 진행
- `--reset`: 작업 상태 초기화 (주의: 모든 진행 상황이 삭제됩니다)
- `--list`: 작성할 문서 목록 확인
- `--next`: 다음 문서 하나만 작성
- `--phase [1-4]`: 특정 Phase의 문서만 작성

## 작업 흐름

1. **상태 확인**: `.snapshot/generate-spec-documents/` 디렉토리에서 작업 상태 로드
2. **다음 문서 식별**: 우선순위와 Phase에 따라 다음 작성할 문서 결정
3. **문서 작성**: 템플릿에 따라 문서 작성
4. **상태 업데이트**: 작업 진행 상황 저장
5. **반복**: 모든 문서 작성 완료까지 반복

## 상태 업데이트 규칙

1. 문서 작성 시작 시:
   - `documents.json`의 해당 문서 `status`를 "in_progress"로 변경
   - `current.json`에 현재 문서 정보 저장
   - `status.json`의 `currentDocument` 업데이트
   - `status.json`의 `statistics.inProgress` 증가, `statistics.pending` 감소

2. 문서 작성 완료 시:
   - `documents.json`의 해당 문서 `status`를 "completed"로 변경
   - `current.json`의 `currentDocumentId`를 null로 설정
   - `status.json`의 `currentDocument`를 null로 설정
   - `status.json`의 `statistics.completed` 증가, `statistics.inProgress` 감소
   - Phase별 및 타입별 통계 업데이트

3. 작업 중단 시:
   - 현재 상태를 그대로 유지
   - `current.json`에 중단 사유 기록 (선택사항)

1. `status.json`과 `documents.json`을 확인하여 현재 상태 파악
2. `current.json`의 `currentDocumentId`가 있으면 해당 문서가 미완성일 수 있음
3. `@generate-spec-documents --continue` 명령어로 작업 재개

## 작업 상태 관리

- `current.json`: 현재 작업 중인 문서 정보
